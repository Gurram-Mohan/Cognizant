Recursion:
Recursion is a programming technique in which a method calls itself to solve a problem by breaking it into 
smaller subproblems. It simplifies problems that have repetitive structures.

Time Complexity:
The recursive algorithm performs one recursive call for each year.
Therefore, the time complexity is O(n), where n is the number of years.

Space Complexity:
The space complexity is O(n) because each recursive call is stored in the call stack.

Optimization:
The recursive solution can be optimized by:
1. Using memoization to store previously computed results.
2. Using an iterative approach to avoid recursive call overhead.
3. Using mathematical formulas for compound growth when applicable.

Conclusion:
Recursion provides a simple and readable solution for financial forecasting, but iterative methods are generally 
more efficient for large inputs.
