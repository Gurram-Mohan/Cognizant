# 🛒 Lab 1 - Understanding ORM with a Retail Inventory System

## 📌 Objective

This lab introduces **Object-Relational Mapping (ORM)** using **Entity Framework Core** by creating the initial setup for a Retail Inventory Management System.

The objective is to understand how Entity Framework Core bridges the gap between C# objects and relational databases while setting up the development environment for future database operations.

---

## 📖 Scenario

A retail store requires an inventory management system to maintain information about products, categories, and stock levels in a SQL Server database.

Entity Framework Core is used as the Object-Relational Mapper (ORM) to simplify database interactions using C# classes instead of writing SQL queries manually.

---

## 🎯 Learning Outcomes

- Understand the concept of Object-Relational Mapping (ORM).
- Learn the advantages of using Entity Framework Core.
- Differentiate between Entity Framework and Entity Framework Core.
- Explore the latest features of EF Core.

- Install the required Entity Framework Core packages.

## 📚 Key Concepts Learned

### Object-Relational Mapping (ORM)

ORM maps C# classes to relational database tables, allowing developers to perform database operations using objects instead of writing SQL queries.

### Benefits of ORM

- Faster Development
- Improved Maintainability
- Reduced SQL Boilerplate
- Strongly Typed Models
- Better Productivity

### Entity Framework Core

Entity Framework Core is Microsoft's lightweight, cross-platform ORM framework that simplifies database development using C#.
